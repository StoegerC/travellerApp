SERVER / BETRIEB (nautilus)
[ ] Persistentes Journal aktivieren — Ursache des Ausfalls vom 10.07.2026
    (~19:40 Uhr offline, Power-Cycle 20:55 Uhr) war nicht mehr feststellbar,
    weil das Journal nur im RAM lag (kein /var/log/journal).
    Fix: sudo mkdir -p /var/log/journal && sudo journalctl --flush
    Optional Größe begrenzen: /etc/systemd/journald.conf.d/size.conf
    mit [Journal] SystemMaxUse=200M, dann systemd-journald neu starten.
[ ] Prüfen ob klassische Logs vom Ausfall existieren:
